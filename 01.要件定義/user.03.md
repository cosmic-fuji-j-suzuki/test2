# ユーザーへの質問・回答  
（`ans.02.md` からの質問への回答。各 `⇒` の後に記入してください）  

G-1. ログインの「グループコード」は、メール一本化に伴い廃止してよいですか？（メールでユーザー・グループを一意特定できるため。QR にはグループ情報を内包）  
  ⇒  一本化しても良いです。  

G-2. 子どものお小遣いは「保護者が付与・記録」する運用ですか？ それとも子ども本人も入力可にしますか？（本人入力なしなら UI がさらに簡単になります）  
  ⇒  複雑ですが、子どもが入力できるようにしたいです。ログインボタン押下後・QR認証後に「ぼく・わたしがにゅうりょく」「ぱぱ・ままがにゅうりょく」で分岐しましょう。  

G-3. 口座管理は「既定 OFF（単一口座でシンプル）／上級者が任意で有効化」で始めてよいですか？（ans.02 の 8.1 推奨）  
  ⇒  「口座管理を既定 OFF（単一口座）」で始める方針でよい  

G-4. 予算は「費目別の月予算」に加えて「総額の月予算（費目横断の上限）」も持たせますか？  
  ⇒  

G-5. 2FA の 6 桁キーの有効期限（例：5〜10分）と、QR の有効期限（例：発行から30日）の目安はいかがしますか？（モックではメール送信はダミー表示）  
  ⇒  例で良いです。が、システム設定テーブルとかで変更できるようにしたいですね。  

G-6. 繰越の実装は「transaction_type に carryover を追加（ans.02 の 8.2 推奨案）」で進めてよいですか？ 期首残高スナップショットは必要時に追加、という段取りでOKですか？  
  ⇒  OKです。  

G-7. 費目プリセット（ans.02 の 5章）に過不足はありますか？（追加したい費目・不要な費目があれば）  
  ⇒  私自身、経理の知識がないので、プリセットとしてはOKだと思います。あとは個別で対応すればよいし、何なら将来テナント間で費目ランキングなど設けて、取り込みできるような機能があると面白そうですね。今はいいです。
