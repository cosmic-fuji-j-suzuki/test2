# 役割  
あなたは熟練のSE・プログラマーです  

# 概要  
このシステムで何を作りたいかを 1〜2 行で書く。  
（例）社内の◯◯を管理する Web アプリを作りたい。  
※ ここはユーザーが追記します。  

# 依頼  
以下のテンプレートを一緒に埋めながら要件を固めたい。  
空欄・曖昧な点は、決めつけずに末尾「# ユーザーへの質問・回答」で質問してほしい。  
埋まっている範囲で、要件定義ドラフトを `# 出力` のファイルにまとめてください。  

---

## 1. 目的・背景  
- 誰が使うか（対象ユーザー・想定人数）:  
- 何を解決したいか（現状の課題）:  
- 今困っていること／うまくいっていないこと:  

## 2. スコープ  
- やること（今回の対象）:  
- やらないこと（今回は対象外）:  

## 3. 機能要件（箇条書きでOK）  
-  
-  

## 4. 非機能要件  
- 想定利用者数 / 同時接続数:  
- 認証・権限（誰が何をできるか）:  
- 対応環境（ブラウザ / OS / PC・スマホ）:  
- 性能・可用性の希望（あれば）:  
- セキュリティ上の要求（あれば）:  

## 5. 画面（分かる範囲で）  
- 画面一覧:  
- 各画面の「表示項目」と「操作」:  
  - （例）一覧画面: 表示＝タイトル/状態、操作＝検索・行クリックで詳細  

## 6. データ項目（画面から逆算すると漏れにくい）  
- （例）ユーザー: id, 氏名, メール, 権限  
-  

## 7. 制約・前提  
- 技術スタックの希望（言語 / フレームワーク / DB。分からなければ「おまかせ」でOK）:  
- 予算・期限:  
- 既存システムや外部サービスとの連携:  
- その他の制約:
