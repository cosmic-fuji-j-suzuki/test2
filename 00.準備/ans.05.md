## 変更内容の詳細  

### 1. 命名規則の定義本文（`## ファイル命名と作業フロー`）  

旧: 依頼書 `NN.<slug>.md`（例 `01.init.md`）／回答書 例 `02.ans.md`  
新: **依頼書 `user.NN.md`（ユーザーのオーダー）／回答書 `ans.NN.md`（AIの回答）** に統一。  
「現在のやり取り」行や参照例もすべて `user.NN.md`→`ans.NN.md` に揃えました。  

### 2. ブランチ運用の簡略化（`## git連携`）  

5種（feature/hotfix/pre-release/main/staging）の詳細記述を、検証段階向けの最小構成へ差し替え:  

- `main` … 常に動く/確定した状態  
- `feature/{NN}-{slug}` … 作業ブランチ（`main` から作成→`main` へマージ、`{NN}` は `user.NN` の番号を流用）  
- `staging` / `hotfix` / `pre-release` は本番デプロイを見据える段階で追加  
- コミットメッセージは Conventional Commits を基本  

### 1-a. 「バックアップ除外対象」→「Git 除外対象（.gitignore の根拠）」  

手動バックアップ廃止・Git移行に合わせ、見出しと導入文を `.gitignore` ベースの記述に修正。  
あわせて「`.gitignore` を言語別に入れ替える際も、`/.trash/` 除外と `.claude/` を除外しない方針は維持する」旨の注意書きを追加しました。  

### 1-b. 行末の半角スペース2個で統一  

散文・箇条書き・見出しの行末を半角スペース2個に統一（Markdown の改行明示）。  
※ フェンスドコードブロック内部は、意味を持たない＆可読性のため付与していません（この方が安全です）。  

### 1-c. 「# ユーザーへの質問・回答」へ統一＋記述者の意味づけ  

見出しを `# ユーザーへの質問・回答` に統一し、CLAUDE.md に次のルールを明記しました:  

- **AI が回答書（`ans.NN.md`）に記述** → ユーザーへの「質問」（回答待ちである旨も明記する）  
- **ユーザーが依頼書（`user.NN.md`）に記述** → 直前の AI の質問への「回答」  

### 2. push の運用ルールを追記（`## git連携 > ### push の運用`）  

テスト push が成功したとのこと、承知しました。今後の運用として CLAUDE.md に以下を明文化:  

- push はユーザーが手動実行（AI はコミットまで。push はパーミッションで拒否されるため）  
- AI は push が必要なタイミングで  
  「`! git push -u origin {作業中のブランチ名}` を実行してください」  
  と案内する（例: `! git push -u origin main`）
